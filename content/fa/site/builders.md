# فلسفه طراحی

FRC به عنوان یک پیکره با شناسه‌های ثابت، تعاریف دقیق و تفکیک واضح بین قوانین (canon) و تفسیر طراحی شده است. این طراحی تضمین می‌کند که ابزارها و عامل‌ها می‌توانند بدون تخریب لایه مرجع، درون چارچوب عمل کنند.

## گردش کار محلی

سورس کد در گیت‌هاب قرار دارد. برای ایشوها و PRها، با شناسه‌های محتوا به عنوان موجودیت‌های تغییرناپذیر رفتار کنید و تغییرات را حداقلی و تکرارپذیر نگه دارید.

```bash
git clone https://github.com/servathadi/fractalresonance
cd frc
npm i
npm run dev
npm run build
```

**نکته:** قبل از کامیت کردن ویرایش‌های محتوا، `npm run content:audit` را اجرا کنید.
